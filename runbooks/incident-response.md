# Incident Response

## Severity Levels

| Level | Description | Response Time |
|-------|-------------|---------------|
| P1 | Production down | Immediate |
| P2 | Major feature broken | < 1 hour |
| P3 | Minor issue, workaround exists | < 4 hours |
| P4 | Low impact / cosmetic | Next sprint |

## Steps

1. Acknowledge the alert
2. Assess severity and assign an owner
3. Create a war-room channel for P1/P2
4. Investigate and apply a fix or rollback
5. Communicate status to stakeholders
6. Write a post-mortem for P1/P2
